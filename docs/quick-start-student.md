# Modo rápido para estudiantes

Esta ruta sirve cuando tienes poco tiempo o cuando quieres practicar antes de leer la explicación completa.

## Regla de trabajo

```text
Ejecutar -> Cambiar una cosa -> Ejecutar -> Explicar
```

Si una lección se siente larga, empieza por la misión rápida y luego vuelve a la teoría.

## Ruta de 60 minutos

| Minutos | Lección | Meta mínima | Archivo |
|---:|---|---|---|
| 0-7 | 01. Primer programa | Cambiar el mensaje visible en consola. | `starter-files/lesson-01-hello-world/Main.java` |
| 7-16 | 02. Variables | Completar nombre, edad, promedio y estado. | `starter-files/lesson-02-variables/Main.java` |
| 16-26 | 03. Condiciones y bucles | Probar intentos con pistas de mayor o menor. | `starter-files/lesson-03-if-loops/Main.java` |
| 26-36 | 04. Métodos | Usar un método para sumar dos números. | `starter-files/lesson-04-methods/Main.java` |
| 36-48 | 05. Clases y objetos | Crear un jugador y sumarle puntos. | `starter-files/lesson-05-classes-objects/` |
| 48-60 | 06. Proyecto final | Ejecutar el juego y explicar una clase. | `starter-files/lesson-06-final-project/` |

## Qué entregar por lección

| Lección | Evidencia corta |
|---|---|
| 01 | Captura de consola con mensaje personalizado. |
| 02 | Ficha impresa con datos modificados. |
| 03 | Dos ejecuciones: una con acierto y una con error corregido. |
| 04 | Resultado de suma y explicación de un método. |
| 05 | Salida con dos jugadores o un jugador con puntaje actualizado. |
| 06 | Partida final y explicación de qué hace `Player` o `Game`. |

## Preguntas antes de avanzar

1. ¿Qué cambiaste?
2. ¿Qué mostró la consola?
3. ¿Qué error apareció o podría aparecer?
4. ¿Qué parte del código no conviene tocar todavía?

Si puedes responder esas cuatro preguntas, la práctica cumplió su objetivo.
